### Basic Authentication Flow

1. **Client sends request** with Authorization header:
   ```
   Authorization: Basic <base64_encoded_credentials>
   ```

2. **Credentials format**: `username:password` encoded in Base64

3. **Server decodes** and validates credentials

4. **Server responds** with requested resource or 401 Unauthorized

## Base64 Encoding

Base64 is a binary-to-text encoding scheme that represents binary data in an ASCII string format.

### Example in Python

```python
import base64

# Encode
credentials = "user@example.com:MyPassword123"
encoded = base64.b64encode(credentials.encode()).decode()
print(encoded)  # dXNlckBleGFtcGxlLmNvbTpNeVBhc3N3b3JkMTIz

# Decode
decoded = base64.b64decode(encoded).decode()
print(decoded)  # user@example.com:MyPassword123
```

## Authentication Concepts

### What is Authentication?

Authentication is the process of verifying the identity of a user or system. It answers the question: "Who are you?"

### What is Basic Authentication?

Basic Authentication is an HTTP authentication scheme where:
- Credentials are sent in the `Authorization` header
- Format: `Authorization: Basic <base64(username:password)>`
- Simple but requires HTTPS for security
- No encryption, only encoding (Base64)

### Security Considerations

⚠️ **Important**: Basic Authentication should only be used over HTTPS because:
- Base64 is encoding, NOT encryption
- Credentials are easily decoded
- Susceptible to man-in-the-middle attacks over HTTP

## Error Handling

- **401 Unauthorized**: Missing or invalid credentials
- **403 Forbidden**: Valid credentials but insufficient permissions
- **404 Not Found**: Resource doesn't exist
